Поля помеченные как transient не будут сериализованны в файл.
Поля помеченные как transient & final то же не будет сериализованны в файл.
Другое дело то, как эти данные будут инициализированы при десериалазации.
- При десериализации по специфике не вызывается дефолтный конструктор, а это
значит что если инициализируете поле как:

private transient final String role = new String("simple_user");

или

private transient final String role;
public User() {
         role = "some"
}

в логах будет null.

В голове нужно держать не то, что поле final transient будет сериализовано в файл, а
то как оно будет инициализировано при десериализации.


----
Десериализация final полей зависит от способа их инициализации:
- при инициализации через конструктор или нестатический блок всё как обычно, десериализация происходит штатно
- но если инициализация происходит при объявлении, то будет использовано значение из инициализации, а значение от десериализации будет проигнорировано
Забавно ещё то, что на serialVersionUID ни значение при инициализации, ни способ инициализации (при объявлении или в нестатическом блоке) не влияет.
